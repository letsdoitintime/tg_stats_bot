# TG Stats Bot Requirements
# Pinned versions for production stability
# Reconciled against the running venv: 2026-07-22
#
# These pins describe what tgstats-bot / tgstats-migrations ACTUALLY run.
# Before this reconciliation the pins were stale by up to a major version, so
# `pip install -r requirements.txt` (Dockerfile, install_requirements.sh) would
# silently DOWNGRADE prod: redis 6.4->5.2, structlog 25.4->24.4, alembic
# 1.16->1.13, psycopg 3.2->3.1, pydantic 2.11->2.10, fastapi 0.116->0.115,
# uvicorn 0.35->0.32, celery 5.5->5.4, pydantic-settings 2.10->2.7.
# Keep this file in step with the venv; a bump belongs here and in the venv together.

# Core dependencies

# the bot writes with. Proving psycopg says nothing about asyncpg.
asyncpg==0.31.*
pydantic==2.11.*
pydantic-settings==2.10.*
# 0.139.x is a MAJOR that DROPPED starlette's upper bound (see the starlette
# comment below — this is the day it started mattering). Killer question for
# a web framework isn't "does it import", it's whether the mechanisms still
# fire: scripts/framework_parity.py builds a small app in-process mirroring
# app.py's own stack — lifespan startup/shutdown (proven via `with
# TestClient`, not just import), two stacked custom middleware layers, CORS
# preflight for an allowed AND a disallowed origin, gzip (and the size
# threshold that skips it), a custom-exception -> JSON handler including MRO
# dispatch (the specific subclass's handler wins over the base-class
# catch-all), the 422 body shape exactly as error_handlers.py wraps it
# (field/message/type extraction from exc.errors()), and dependency_overrides.
# 16 checks, byte-identical 0.116.1 -> 0.139.2. The full tests/ suite was also
# re-run under the new pins: same 234 passed / 3 pre-existing failures
# (unrelated settings-monkeypatch tests, fail identically on both versions).
fastapi[all]==0.139.*
# starlette is the mechanism layer fastapi drives (middleware stack, CORS,
# gzip, exception dispatch) — proven on the SAME harness as fastapi above,
# since bumping one without the other was never possible to test in
# isolation. One new, harmless signal: starlette.testclient now prefers an
# optional `httpx2` package and warns when falling back to plain httpx
# (still installed, still what TestClient actually uses) — not a behaviour
# change, nothing to act on now.
starlette==1.3.*
# 0.51.x: TestClient's ASGITransport never touches uvicorn's real HTTP
# server loop, so framework_parity.py says nothing about this package on its
# own. Separately smoke-tested with a real uvicorn.Server on a real socket
# (not TestClient) serving one request end-to-end: identical status, body,
# and headers on 0.35.0 and 0.51.0.
uvicorn==0.51.*
# 26.x is a MAJOR, and structlog is imported by nearly every module here.
# Killer question was whether the configured chain still RENDERS the same —
# no test reads log lines, so a reordered key or changed exception folding
# would ship silently. scripts/structlog_parity.py renders a fixed event set
# through 4 chains, including the app's own ColoredConsoleRenderer and
# add_app_context: 35 lines, byte-identical across 25.4.0 -> 26.1.0.
structlog==26.1.*
emoji==2.14.*
# 1.2.x verified to parse the real .env, .env.example and 11 syntax edge cases

# to 1.1.x. Reached via pydantic-settings, which is what actually loads config.
